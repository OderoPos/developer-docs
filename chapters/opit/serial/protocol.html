Communication between the ECR and the POS terminal follows the Hypercom binary protocol. Every message is a raw byte sequence with a well-defined structure.

Frame format:

STX | MessageID (3 bytes) | MessageType (2 bytes) | Separator | Data | ETX | LRC
Field Size Value / Description
STX 1 byte 0x02 — Start of Text
MessageID 3 bytes Operation identifier: MOL, VOI, or STL
MessageType 2 bytes Direction/phase code: 10, 11, 12, or 13
Separator 1 byte 0x2E (.)
Data Variable Payload — see per-message definitions below
ETX 1 byte 0x03 — End of Text
LRC 1 byte Longitudinal Redundancy Check — XOR of all bytes from (exclusive) STX to ETX (inclusive)

Control bytes (single-byte messages):

  • ACK — 0x06: Positive acknowledgment.
  • NAK — 0x15: Negative acknowledgment.

Message types (phase codes):

Code Direction Description
10 ECR → POS InitTransaction — request to start an operation
11 POS → ECR InitResponse — acknowledgment of the request
12 POS → ECR AuthResult — outcome of the operation
13 ECR → POS AuthResponse — acknowledgment of the result

LRC calculation:

XOR all bytes strictly between STX and ETX, then XOR the result with ETX (i.e., ETX is included in the checksum but STX is not).
