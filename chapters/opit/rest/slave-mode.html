The Slave Mode feature within the OPit API allows integrators to control the interaction with the terminal after it has been connected to the system. When this feature is activated, the POS terminal displays a locked screen without any interactive elements, preventing direct manipulation of the device until Slave Mode is deactivated. The primary purpose of this feature is to prevent unauthorized or accidental manipulation of the terminal while it is connected to and managed by an external system.

Communication with the Slave Mode API is achieved by sending a structured JSON object via a POST request to the designated endpoint on the device.

The request model is as follows:

    POST: http://{{deviceIP}}:9292/command/slave-mode
{
  "requestId": "rq_unique_identifier",
  "callBack": null,
  "enable": true,
  "title": "Screen title text",
  "message": "Screen message text"
}

Request Parameters:

  • requestId (string, mandatory): A unique client-generated identifier for each request. This is used to correlate responses with requests or for logging and auditing purposes. Example: "rq_101", "slave_mode_001".

  • callBack (string or null, optional): A URL or identifier to which the OPit system might send an asynchronous notification regarding the processing of the command. If no callback is needed, this field can be null or omitted, depending on the API specifications.

  • enable (boolean, mandatory): A flag that controls whether the slave mode feature is activated or deactivated on the fiscal terminal.

    • true: Activates slave mode, enabling the terminal to display the provided title and message.
    • false: Deactivates slave mode, returning the terminal to its default operational state.

  • title (string, optional): The title text to be displayed on the fiscal terminal's screen. This parameter is most relevant when enable is set to true. The title provides context or identifies the current state or purpose. Example: "Ready for Payment", "Waiting for Confirmation".

  • message (string, optional): The main message text to be displayed on the fiscal terminal's screen. This parameter is most relevant when enable is set to true. The message typically provides instructions, status information, or user guidance. Example: "Please wait while processing...", "Transaction completed successfully".

Response Model:

{
    "requestId": "rq_101",
    "success": true,
    "extras": "Slave request handled."
}

Response Parameters:

  • requestId (string): The unique identifier that was provided in the corresponding request. This allows clients to match responses with their requests, especially important when handling asynchronous operations or batch processing.

  • success (boolean): A flag indicating whether the slave mode command was executed successfully.

    • true: The command was processed successfully, and the slave mode has been configured as requested.
    • false: The command encountered an error during processing, and the slave mode configuration was not applied.

  • extras (string): Additional information or status message returned by the server. Typical responses include "Slave request handled." to confirm successful processing, or error descriptions if the operation failed.
