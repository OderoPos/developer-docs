The Fiscal Export endpoint allows integrators to export fiscal data (XML reports or electronic journal) directly from the fiscal device. Unlike standard fiscal command endpoints, this endpoint returns a binary ZIP stream instead of a JSON payload.

Communication with the Fiscal Export API is achieved by sending a structured JSON object via a POST request to the dedicated export endpoint.

The request model is as follows:

    POST: http://{{deviceIP}}:9292/command/ecr/fiscal/export
{
  "requestId": "rq_unique_identifier",
  "fiscalCommand": {
    "report": {
      "type": "XMLByDate",
      "startDate": "113200100326",
      "stopDate": "112800110326"
    }
  }
}

Common Request Parameters:

  • requestId (string, mandatory): A unique client-generated identifier used for tracing and logging. Example: "rq_101".
  • fiscalCommand (object, mandatory): Container for the export instruction. It must include either a report object or a command object, depending on the export type.

Endpoint Details:

  • URL: /command/ecr/fiscal/export
  • Method: POST
  • Content-Type: application/json
    Request:
{
  "requestId": "rq_101",
  "fiscalCommand": {
    "report": {
      "type": "XMLByDate",
      "startDate": "113200100326",
      "stopDate": "112800110326"
    }
  }
}

Command: XML Report by Date

Description: Exports a fiscal XML report for a specific date/time range.

Parameters:

  • type (string, mandatory): Must be "XMLByDate".
  • startDate (string, mandatory): Start date/time in format HHmmDDMMYY. Example: "113200100326".
  • stopDate (string, mandatory): End date/time in format HHmmDDMMYY. Example: "112800110326".



    Request:
{
  "requestId": "rq_101",
  "fiscalCommand": {
    "report": {
      "type": "XML",
      "startZ": "1",
      "stopZ": "3"
    }
  }
}

Command: XML Report by Z

Description: Exports a fiscal XML report for a Z report interval.

Parameters:

  • type (string, mandatory): Must be "XML".
  • startZ (string, mandatory): First Z report number in the interval.
  • stopZ (string, mandatory): Last Z report number in the interval.



    Request:
{
  "requestId": "rq_101",
  "fiscalCommand": {
    "command": {
      "cmdCode": "JournalExportByDate",
      "startDate": "113200100326",
      "stopDate": "115800110326"
    }
  }
}

Command: Electronic Journal by Date

Description: Exports the electronic journal for a date/time range.

Parameters:

  • cmdCode (string, mandatory): Must be "JournalExportByDate".
  • startDate (string, mandatory): Start date/time in format HHmmDDMMYY.
  • stopDate (string, mandatory): End date/time in format HHmmDDMMYY.



    Request:
{
  "requestId": "rq_101",
  "fiscalCommand": {
    "command": {
      "cmdCode": "JournalExport",
      "param1": "1",
      "param2": "3"
    }
  }
}

Command: Electronic Journal by Z

Description: Exports the electronic journal for a Z report interval.

Parameters:

  • cmdCode (string, mandatory): Must be "JournalExport".
  • param1 (string, mandatory): First Z report number in the interval.
  • param2 (string, mandatory): Last Z report number in the interval.



Example cURL Request:

curl -X POST http://172.30.23.251:9292/command/ecr/fiscal/export \
  -H "Content-Type: application/json" \
  -d '{
    "requestId": "rq_101",
    "fiscalCommand": {
      "report": {
        "type": "XML",
        "startZ": "1",
        "stopZ": "2"
      }
    }
  }' \
  --output /path/to/output/export.zip

Important Notes:

  • requestId is used for tracing and does not influence the export result.
  • Date/time format for startDate and stopDate is HHmmDDMMYY (for example 113200100326 means 11:32, 10 March 2026).
  • This endpoint is long-running and does not enforce a short request timeout. The client should use a sufficiently large read timeout.
  • Replace {{deviceIP}} with the actual IP address of the Android terminal. The API port is always 9292.
