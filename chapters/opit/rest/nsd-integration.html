The Opit REST application runs an NSD (Network Service Discovery) service on Android devices. This service is automatically available whenever the device has an active network connection with a valid IP address. Integrators can use NSD to discover the device on the local network and communicate with the REST API server.

Service Availability

When the Service Runs

The NSD service is active under the following conditions:

  • The Android device has an active network connection (WiFi, Ethernet, USB Tethering, or other network interface)
  • The device has obtained a valid IP address on that network
  • The Opit REST application is running and the LocalServerService is started

The NSD service automatically:

  • Registers when a network becomes available and an IP address is assigned
  • Re-registers when the device switches networks (e.g., from WiFi to USB Tethering)
  • Unregisters when the device loses network connectivity

Service Details

  • Service Type: _http._tcp.
  • Service Name Format: opit_rest_api_<DEVICE_ID>
    • Example: opit_rest_api_PT03D34M00170
  • Port: 9292
  • Protocol: HTTP

Discovering the NSD Service

Example: Once a service is found, resolve it to obtain the host IP address and port.

After resolving an opit_rest_api_PT03D34M00170 service:

  • IP Address: 172.30.23.251
  • Port: 9292
  • Server URL: http://172.30.23.251:9292

Once you have discovered the device and obtained its IP address and port, you can communicate with the REST API server using standard HTTP requests.

Important Considerations

Network Stability

  • The device may switch networks (e.g., WiFi to USB Tethering), causing the IP address to change
  • When a network switch occurs, the NSD service will re-register with the new IP address
  • Integrators should re-discover the service if connection attempts fail

Same Network Requirement

  • The device running the Opit REST application and the integrating application must be on the same local network
  • NSD discovery will not work across different networks or the internet
  • Both devices must support mDNS (Multicast DNS), which is standard on modern Android devices

Service Not Discovered

If you cannot discover the service, check the following:

  1. Verify Network Connectivity: Ensure both devices are on the same local network
  2. Check IP Address: Confirm the device has obtained a valid IP address
  3. Verify Service is Running: Ensure the Opit REST application is running on the device
  4. Restart Discovery: Stop and restart the device

Connection Refused

If connection attempts are refused, verify the following:

  1. Verify IP and Port: Double-check the discovered IP address and port (9292)
  2. Check Firewall: Ensure no firewall is blocking port 9292
  3. Verify HTTP Protocol: Use HTTP, not HTTPS, for local API calls

Service Lost After Connection

This can occur when the device switches networks. Restart the NSD discovery to find the service with its new IP address.

Summary

The Opit REST application provides an NSD-discoverable service that allows integrators to:

  1. Automatically discover the device on the local network
  2. Identify the device's IP address
  3. Communicate with the REST API server on port 9292

By following the integration steps in this guide, integrators can build robust, discovery-based communication with Opit REST devices without requiring manual IP configuration.
