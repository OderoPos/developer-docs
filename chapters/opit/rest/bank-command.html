The bank commands functionality allows an external application to interact with the banking application on the POS terminal to perform various non-transactional operations, such as end-of-day settlement, printing reports, or reprinting receipts.

Communication with the API for bank commands is achieved by sending a structured JSON object. These commands are executed by making a POST request to the following endpoint on the device:

http://{{deviceIP}}:9292/command/bank

Base Request Structure:

All bank commands share a base request structure, to which specific fields are added depending on the desired operation.

    POST: http://{{deviceIP}}:9292/command/bank
{
  "requestId": "unique_request_id",
  "callBack": "URL_for_callback_or_null",
  "command": "specific_bank_command_name",
  // ... other command-specific fields
}

Common Request Parameters:

  • requestId (string, mandatory): A unique client-generated identifier for each request. This can be used to correlate responses with requests. Example: "rq_bank_001", "op_12345".
  • callBack (string, optional): A URL or another identifier to which the system might send an asynchronous notification regarding the command processing. If no callback is needed, this field can be null or an empty string ("").
  • command (string, mandatory): The specific name of the bank command to be executed. Possible values are described below for each command.

Command: Settlement

{
  "requestId": "rq_settlement_789",
  "callBack": "",
  "command": "settlement"
}

Command: settlement

Description: Initiates the end-of-day settlement process on the POS terminal.

Sample Request Body:

Specific Request Parameters:

  • There are no additional specific parameters for this command.

Command: Last Settlement

{
  "requestId": "rq_last_settlement_101",
  "callBack": null,
  "command": "last_settlement",
  "currencyCode": 946
}

Command: last_settlement

Description: Requests information about or reprinting of the last end-of-day settlement report.

Sample Request Body:

Specific Request Parameters:

  • currencyCode (integer, optional): The numeric ISO 4217 currency code. The default value is 946 (RON). Example: 978 for EUR.

Command: Simple X Report

{
  "requestId": "rq_simple_x_rep_102",
  "callBack": "",
  "command": "simple_x_report",
  "currencyCode": 946,
  "totalsOnly": true
}

Command: simple_x_report

Description: Generates a simplified X report (control report). By default, it displays totals only.

Sample Request Body:

Specific Request Parameters:

  • currencyCode (integer, optional): The numeric ISO 4217 currency code. The default value is 946 (RON).
  • totalsOnly (boolean, optional): If true, the report will only contain totals. The default value is true.

Response Structure (for all bank commands):

    Successful Response:
{
    "requestId": "the_id_sent_in_request",
    "success": true,
    "extras": "Operation completed successfully."
}

    Error Response:
{
    "requestId": "the_id_sent_in_request",
    "success": false,
    "extras": "Error message describing the issue."
}

The response will indicate the success or failure of the requested operation.

Response Parameters:

  • requestId (string): The unique identifier that was sent in the request, used for correlation.
  • success (boolean): Indicates the outcome of the operation.
    • true: The operation was processed successfully (or the request to process it was successfully initiated).
    • false: Processing failed or an error occurred.
  • extras (string): Contains additional information about the operation.
