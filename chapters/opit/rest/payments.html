The ProcessPayment command is used to initiate a payment transaction on the POS terminal. This allows an external application to request a payment for a specified amount, and receive a response indicating the outcome of the transaction.

Request Body Structure:

{
  "requestId": "rq_101",
  "callBack": "",
  "amount": 3.5,
  "shouldPrintReceipts": false,
  "currencyCode": 946
}

Command: ProcessPayment

Description: Initiates a payment process on the POS terminal for the given amount and currency. The command allows specifying whether receipts should be printed by the terminal.

Request Parameters:

  • requestId (string, mandatory): A unique client-generated identifier for each request. This can be used to correlate responses with requests. Example: "rq_101", "payment_tx_77234".
  • callBack (string, optional): A URL or another identifier to which the system might send an asynchronous notification regarding the processing of the payment. If no callback is needed, this field can be null or an empty string ("").
  • amount (number, mandatory): The monetary value of the transaction to be processed. This should be a numeric value representing the amount in the specified currency. Example: 3.5, 100.00, 50.
  • shouldPrintReceipts (boolean, optional): Indicates whether the POS terminal should print the transaction receipts. If omitted, the default value is false.
    • true: The POS terminal will attempt to print the receipts.
    • false: The POS terminal will not print the receipts (or follow its default behavior if printing is mandatory for certain transaction types).
  • currencyCode (integer, optional): The numeric ISO 4217 currency code for the transaction amount. If omitted, the default value is 946 (RON - Romanian Leu). Example: 978 for EUR, 840 for USD.

Response Body Structure:

The response will indicate the success or failure of the payment operation.

    Successful response:
{
    "requestId": "rq_101",
    "success": true,
    "extras": "{\"receiptNumber\":\"0\",\"bankActivityResultCode\":\"-1\",\"bankStatus\":\"0\"}"
}

    Error Response:
{
    "requestId": "rq_101",
    "success": false,
    "extras": "Error message"
}

Response Parameters:

  • requestId (string): The unique identifier that was sent in the request, used for correlation.
  • success (boolean): Indicates the outcome of the payment operation.
    • true: The payment was processed successfully (or the request to process it was successfully initiated, depending on the exact flow).
    • false: The payment processing failed or an error occurred.
  • extras (string): Contains additional information about the transaction.
    • On success: This may be a JSON string containing details from the payment processor or bank, such as receipt numbers, authorization codes, or status codes. The exact content can vary. Example: "{\"receiptNumber\":\"0\",\"bankActivityResultCode\":\"-1\",\"bankStatus\":\"0\"}". It is the client's responsibility to parse this JSON string if needed.
    • On error: This typically contains a human-readable error message describing the issue. Example: "Insufficient funds", "Card declined", "Terminal busy".
