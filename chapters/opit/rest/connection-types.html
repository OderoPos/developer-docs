To integrate with the OPit fiscal terminal and use the REST API, you need to establish a network connection between your application and the device. The OPit terminal supports two primary connection methods: WiFi and USB (via USB Tethering). Once connected, you will obtain an IP address that will be used to make REST API calls to the device.

Available Connection Methods:

WiFi Connection

The WiFi connection method is the most straightforward way to integrate with the OPit terminal. This approach is ideal for wireless environments and provides flexibility in device placement.

Setup Steps:

  1. Connect the OPit device to your WiFi network:

    • Navigate to the device's WiFi settings
    • Select your WiFi network from the available networks list
    • Enter the WiFi password if required
    • Wait for the device to establish the connection

  2. Obtain the device IP address:

    • Once connected, the device will receive an IP address from your network's DHCP server
    • You can find the IP address in the device's network settings or system information screen
    • The IP address will typically be in the format: 192.168.x.x or 10.x.x.x depending on your network configuration

  3. Test the connection:

    • Use the obtained IP address to make REST API calls to the device
    • Example: http://192.168.1.100:9292/
    • Ensure your application and the OPit device are on the same network or that network routing is properly configured

USB Connection (USB Tethering)

The USB connection method uses USB Tethering to create a direct network connection between the OPit terminal and your system. This method is ideal for scenarios where WiFi is unavailable or when a more stable, dedicated connection is required.

Setup Steps:

  1. Connect the OPit device to your system via USB cable:

    • Use a compatible USB cable to physically connect the OPit terminal to your system
    • Ensure the USB cable supports data transfer (not charge-only cables)

  2. Enable USB Tethering on the OPit device:

    • Open the device Settings menu
    • Navigate to Network & Internet (or Connections, depending on the Android version)
    • Select Hotspot & Tethering (or Mobile Hotspot and Tethering)
    • Enable the USB Tethering option by toggling the switch
    • You should see a notification confirming that USB tethering is active

  3. Verify the network connection:

    • Once USB tethering is enabled, a local network will be created between your system and the OPit device
    • Your system will automatically receive an IP address from the local network created by the device

  4. Identify the device IP address:

    • Check your system's network settings to identify the new network interface created by USB tethering
    • When the local server is active, the device screen displays a notification indicating this and the server IP address.

  5. Test the connection:

    • Use the obtained IP address to make REST API calls to the device
    • Example: http://192.168.42.129:9292/
    • Verify that your system can successfully communicate with the device

Important Notes:

  • Port Number: All REST API calls are made to port 9292 regardless of the connection method used.

  • Network Requirements: For WiFi connections, ensure that your network allows communication between devices (some networks have client isolation enabled which prevents device-to-device communication).

  • USB Driver: For USB tethering on Windows systems, you may need to install appropriate USB drivers for the Android device. Most modern operating systems will automatically install the required drivers.

  • Firewall: Ensure that your system's firewall allows outbound connections to the device's IP address on port 9292.

  • Connection Stability: USB tethering generally provides a more stable connection compared to WiFi, especially in environments with WiFi interference.

API Usage:

Once you have established a connection and obtained the device's IP address, you can start making REST API calls. All commands will be sent as JSON objects to the device's REST endpoints. The specified date and numeric formats must be strictly adhered to. For additional information and example API calls, you can download this Postman collection.
