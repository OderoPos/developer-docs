The Sale command within the OPit API is fundamental for processing sale transactions and issuing fiscal receipts. It allows external applications, such as POS (Point of Sale) systems, to transmit complete details of a sale to the fiscal terminal for recording and printing. By using this command, developers can ensure a smooth and fiscally compliant integration, automating the sale process.

    // For example: "dateTime": { "value": "..." } for SetDateTime
    // Or: "sale": { ... } for a Sale transaction
    }
}

Communication with the API for fiscal commands is achieved by sending a structured JSON object. All requests to the fiscal command endpoint share a common base structure, which includes a unique request identifier, an option for a callback, and a dedicated object for the actual fiscal command.

Common Data Types and Key Concepts

Before detailing each command, here are some common data types and concepts used in the API:

  • Report Output (output): This parameter, present in many reporting commands, defines where the report result will be sent. The primary value is:

    • "Printer": Sends the report to the fiscal printer.

  • Report Size (size): Specifies the level of detail for certain reports. Possible values are:

    • "Compressed": Generates a summary report containing only essential information.
    • "Extended": Generates a detailed report, including all available information for that report type.

  • Position of Free Text on Receipt (position in freeTexts for Sale): Indicates where free text will be printed on the fiscal receipt. Possible values are:

    • "head": The text will be added to the header (top part) of the receipt.
    • "foot": The text will be added to the footer (bottom part) of the receipt.

  • Date/Time Format: All date and time fields must adhere to the HHmmssddMMyy format.

    • HH: Hour (00-23)
    • mm: Minutes (00-59)
    • ss: Seconds (00-59)
    • dd: Day (01-31)
    • MM: Month (01-12)
    • yy: Year (last two digits, e.g., 23 for 2023)
    • Example: 143000231023 represents October 23, 2023, 02:30:00 PM.

  • Numeric Format (Mandatory for Amounts/Values): Numbers representing monetary values (prices, amounts, discounts, adjustments) or quantities with decimals must be provided as strings, using a comma (,) as the decimal separator. The number of decimal places may vary depending on the context (e.g., 2 decimal places for prices, 3 for quantities).

    • Mandatory Example: "100,00", "1,500", "25,75".
    • DO NOT use a period (.) as a decimal separator in these strings.

Communication with the API for fiscal commands is achieved by sending a structured JSON object. All fiscal command executions are performed by making a POST request to the following endpoint on the device:

http://{{deviceIP}}:9292/command/ecr/fiscal

The {{deviceIP}} placeholder should be replaced with the actual IP address of the target POS device.

All requests to this endpoint share a common base JSON structure in the request body, which includes a unique request identifier, an option for a callback, and a dedicated object for the actual fiscal command.

The basic request body model is as follows:

    POST: http://{{deviceIP}}:9292/command/ecr/fiscal
{
  "requestId": "rq_unique_identifier",
  "callBack": "url_or_identifier_for_callback_if_any",
  "fiscalCommand": {
    // The specific fiscal command object will be included here
    // For example: "dateTime": { "value": "..." } for SetDateTime
    // Or: "sale": { ... } for a Sale transaction
    }
}
  • requestId (string, mandatory): A unique client-generated identifier for each request. This can be used to correlate responses with requests or for logging purposes. Example: "rq_101", "sale_tx_55789".
  • callBack (string, optional): A URL or another identifier to which the OPit system might send an asynchronous notification regarding the processing of the command. If no callback is needed, this field can be an empty string ("") or omitted, depending on the API specifications.
  • fiscalCommand (object, mandatory): A JSON object containing the specific details of the fiscal command to be executed. The internal structure of this object varies depending on the fiscal command being called (e.g., SetDateTime, Sale, ZReport, etc.), as detailed in the following sections for each command.

Command: Sale

Description: The command is represented by a JSON object with a single main key, sale. The value of this key is another object containing all transaction parameters.

Parameters:

  • sysInfo (object, mandatory): Contains information about the operator and the point of sale.

    • posNo (string, mandatory): Identifier of the point of sale or cash register (e.g., "1", "A02").
    • cashier (string, mandatory): Name of the operator/cashier processing the transaction (e.g., "John Doe").

  • custInfo (object, optional): Contains customer information.

    • cui (string, optional): The customer's Unique Registration Code (CUI) or Fiscal Identification Code (CIF), if applicable (e.g., "12345678").

  • products (array of objects, mandatory): Represents the list of items sold. Each object in the array describes an individual product.

    • name (string, mandatory): Name of the product (e.g., "Bread", "Milk UHT 1.5%").
    • um (string, mandatory): Unit of measure for the product (e.g., "pcs", "kg", "L").
    • quantity (string, mandatory): Quantity sold. Numeric format, with a comma as decimal separator (e.g., "2,00", "1,5", "0,750").
    • price (string, mandatory): Unit price of the product (excluding VAT, if VAT is applied separately). Numeric format, with a comma as decimal separator (e.g., "5,00", "7,50", "100,99").
    • vatId (string, mandatory): Identifier of the VAT rate applicable to the product (e.g., "A", "B"). Must correspond to a predefined VAT rate in the fiscal system.
    • discountType (string, optional): Type of discount or surcharge applied individually to the item.
      • "0": Value-based (fixed amount).
      • "1": Percentage-based (calculated from the item's value).
    • discountValue (string, optional, mandatory if discountType is present): Value of the discount or surcharge. Numeric format, with a comma as decimal separator. Use a minus sign ("-") for discounts (e.g., "-1,00", "10,00" for 10%).
    • specialTaxName (string, optional): Name or type of any special tax applicable to this item (e.g., "Excise Duty", "EcoTax"). This text will be printed on the receipt alongside the tax value.
    • specialTaxValue (string, optional, mandatory if specialTaxName is present): The monetary value of the special tax for the item. Numeric format, with a comma as decimal separator (e.g., "0,50", "2,25"). This value is typically added to the item's price.

  • paymentMethods (array of objects, mandatory): Describes the payment methods used to pay for the transaction. The sum of values from all payment methods must cover the grand total of the receipt.

    • paymentMethodId (integer, mandatory): A numeric identifier for the payment type.
      • 0 for Cash
      • 3 for Card
      • 1 for Check
      • 1 for Voucher
      • 2 for Credit
    • value (string, mandatory): Amount collected through this payment method. Numeric format, with a comma as decimal separator (e.g., "20,25", "150,00").

  • freeTexts (array of objects, optional): Allows adding custom text lines to the fiscal receipt.

    • freeText (string, mandatory): The text to be printed (e.g., "Thank you for your purchase!", "Promotion valid until X date").
    • position (string, mandatory): Specifies where the text will be printed on the receipt. Possible values: "head" (in the header), "foot" (in the footer).

  • totDiscount (array of objects, optional): Defines discounts or surcharges applied to the grand total of the receipt, possibly differentiated by VAT rates.

    • vatId (string, mandatory): Identifier of the VAT rate to which this adjustment applies (e.g., "A").
    • discountType (integer or string, mandatory): Type of adjustment.
      • 0 or "0": Value-based.
      • 1 or "1": Percentage-based (calculated from the total corresponding to the specified VAT rate).
    • discountValue (string, mandatory): Value of the adjustment. Numeric format, with a comma as decimal separator. Use a minus sign ("-") for discounts (e.g., "-0,50").

  "cashOperation": {
    "operationType":"Current",
    "amount": "100,00",
    "description": "Depunere numerar"
  }
}

Command: CashOperation

Description: Registers a cash deposit (IN) or withdrawal (OUT) operation.

Parameters:

  • type (string, mandatory): Type of cash operation.
    • "Initial": Initial cash deposit at the beginning of the day/shift.
    • "Current": Cash deposit.
  • amount (string, mandatory): The amount of cash. Numeric format with comma decimal separator (e.g., "100,00").
  • description (string, mandatory): A brief description or reason for the operation.

Command: NonFiscalReceipt

Description: Prints a non-fiscal receipt, which does not affect fiscal totals. Useful for various informational printouts.

Parameters:

  • textLines (array of objects, mandatory): Lines of text to be printed. Each object has:
    • text (string, mandatory): The text content for the line (max 48 characters, but printer-dependent).
    • printAttributes (string, optional): See "Common Data Types". Comma-separated attributes like "CENTER", "BOLD".
  • hasHeader (boolean, optional, default: false): If true, prints the standard fiscal header (company info, etc.) before the custom text.
  • hasFooter (boolean, optional, default: false): If true, prints the standard fiscal footer (fiscal series, etc.) after the custom text.
