Fiscal Commands API Documentation

This API allows interaction with the fiscal memory by sending commands as JSON objects.

General Note: All commands will be sent as a JSON object. The specified date and numeric formats must be strictly adhered to.

Common Data Types and Key Concepts

Before detailing each command, here are some common data types and concepts used in the API:

  • Report Output (output): This parameter, present in many reporting commands, defines where the report result will be sent. The primary value is:

    • "Printer": Sends the report to the fiscal printer.

  • Report Size (size): Specifies the level of detail for certain reports. Possible values are:

    • "Compressed": Generates a summary report containing only essential information.
    • "Extended": Generates a detailed report, including all available information for that report type.

  • Position of Free Text on Receipt (position in freeTexts for Sale): Indicates where free text will be printed on the fiscal receipt. Possible values are:

    • "head": The text will be added to the header (top part) of the receipt.
    • "foot": The text will be added to the footer (bottom part) of the receipt.

  • Non-Fiscal Text Attributes (printAttributes in NonFiscalReceipt.textLines): Specifies the formatting of text lines on a non-fiscal receipt. This is a string that can contain one or more comma-separated values, corresponding to print attributes. Possible values for each attribute are:

    • "CENTER": Centers the text.
    • "BOLD": Prints the text in bold characters.
    • "DoubleHeight": Prints the text with double height.
    • "DoubleWidth": Prints the text with double width.
    • Example: "BOLD,CENTER" for bold and centered text. The exact implementation and availability of attributes may depend on the printer model.

  • Date/Time Format: All date and time fields must adhere to the HHmmssddMMyy format.

    • HH: Hour (00-23)
    • mm: Minutes (00-59)
    • ss: Seconds (00-59)
    • dd: Day (01-31)
    • MM: Month (01-12)
    • yy: Year (last two digits, e.g., 23 for 2023)
    • Example: 143000231023 represents October 23, 2023, 02:30:00 PM.

  • Numeric Format (Mandatory for Amounts/Values): Numbers representing monetary values (prices, amounts, discounts, adjustments) or quantities with decimals must be provided as strings, using a comma (,) as the decimal separator. The number of decimal places may vary depending on the context (e.g., 2 decimal places for prices, 3 for quantities).

    • Mandatory Example: "100,00", "1,500", "25,75".
    • DO NOT use a period (.) as a decimal separator in these strings.

Commands and Parameters

Below is a list of available commands and their specific parameters. Each command should be sent as the root of a JSON object.

Example Command Structure:

{
"SetDateTime": {
"dateTime": "100000010124"
}
}

Command: SetDateTime

Description: Sets the date and time of the fiscal memory.

Parameters:

  • dateTime (string, mandatory): The date and time to set. Format: HHmmssddMMyy.
{
"SetCompanyData": {
"name": "My Company SRL",
"cui": "RO12345678",
"address": "Str. Exemplu Nr. 10",
"fiscalSeries": "XYZ"
}
}

Command: SetCompanyData

Description: Sets the company data for the fiscal memory.

Parameters (all are strings and mandatory):

  • name: Company name (max 36 characters).
  • cui: Unique company identifier (max 16 characters).
  • address: Company address (max 36 characters).
  • fiscalSeries: Fiscal series identifier (max 8 characters).
{
"GetCompanyData": {}
}

Command: GetCompanyData

Description: Retrieves the company data from the fiscal memory.

Parameters: None.

{
"SetVATRates": {
"vatList": [
{"vatId":"A","vatRate":"19,00"},
{"vatId":"B","vatRate":"9,00"},
{"vatId":"C","vatRate":"5,00"},
{"vatId":"I","vatRate":"0,00"}
]
}
}

Command: SetVATRates

Description: Sets the VAT (Value Added Tax) rates for the fiscal memory.

Parameters:

  • vatList (array of objects, mandatory): A list of VAT objects to set. Each object has:
    • vatId (string, mandatory): The VAT identifier (e.g., "A", "B", "C", "D", "E", "F", "G", "H", "I"). The ID "A" is typically for the standard VAT rate, "B" for the first reduced rate, etc. "I" is usually for VAT exempt items.
    • vatRate (string, mandatory): The VAT rate as a percentage, using a comma as decimal separator (e.g., "19,00" for 19%, "9,00" for 9%, "0,00" for exempt). Max 5 characters including comma.
{
"GetVATRates": {}
}

Command: GetVATRates

Description: Retrieves the VAT rates from the fiscal memory.

Parameters: None.

{
"Fiscalize": {}
}

Command: Fiscalize

Description: Initializes the fiscalization process for the fiscal memory. This is a one-time operation for a new fiscal device.

Parameters: None.

{
"InitElectronicJournal": {}
}

Command: InitElectronicJournal

Description: Initializes the electronic journal of the fiscal memory.

Parameters: None.

{
"TechnicalReport": {
"output": "Printer"
}
}

Command: TechnicalReport

Description: Generates a technical report from the fiscal memory. This report usually contains device status, configuration, and diagnostic information.

Parameters:

  • output (string, mandatory): See "Common Data Types". Typically "Printer".
{
"ZReport": {
"output": "Printer"
}
}

Command: ZReport

Description: Generates the Z report, which is the daily fiscal closure. This report finalizes the day's transactions and resets daily totals.

Parameters:

  • output (string, mandatory): See "Common Data Types". Typically "Printer".
{
"XReport": {
"output": "Printer"
}
}

Command: XReport

Description: Generates the X report, an interim fiscal report showing current day totals without closing the fiscal day.

Parameters:

  • output (string, mandatory): See "Common Data Types". Typically "Printer".
{
"InitJournal": {
"output": "Printer"
}
}

Command: InitJournal

Description: Generates a report about the initialization of the electronic journal.

Parameters:

  • output (string, mandatory): See "Common Data Types". Typically "Printer".
{
"Journal": {
"startDate": "011023",
"endDate": "231023",
"output": "Printer"
}
}

Command: Journal

Description: Generates a journal report for a specified date range.

Parameters:

  • startDate (string, mandatory): The start date for the report. Format: ddMMyy.
  • endDate (string, mandatory): The end date for the report. Format: ddMMyy.
  • output (string, mandatory): See "Common Data Types". Typically "Printer".
{
"JournalExport": {
"startZNo": 100,
"endZNo": 150,
"output": {
"File": {
"exportPath": "/root/Docs/Fiscal/Export/JOURNAL.TXT"
}
}
}
}

Command: JournalExport

Description: Exports journal data for a specified range of Z report numbers. This command is usually used for exporting data to external storage or systems, not directly to the printer.

Parameters:

  • startZNo (integer, mandatory): The starting Z report number for the export.
  • endZNo (integer, mandatory): The ending Z report number for the export.
  • output (object, mandatory): Specifies the export path.
    • File (object):
      • exportPath (string, mandatory): The full path where the export file will be saved (e.g., "/mnt/usb_storage/journal_export.txt"). The exact path format depends on the device's operating system and available storage.
{
"XJournal": {
"output": "Printer"
}
}

Command: XJournal

Description: Generates an XJournal report, which is an interim (non-closing) report of the electronic journal's current state.

Parameters:

  • output (string, mandatory): See "Common Data Types". Typically "Printer".

Command: Sale

Description: Processes a sale transaction, printing a fiscal receipt.

Parameters:

  • sellerInfo (object, mandatory): Information about the seller/cashier.
    • cashierName (string, mandatory): Name of the cashier (max 36 characters).
    • posNumber (string, mandatory): Point of Sale (POS) or department identifier (max 3 characters).
  • products (array of objects, mandatory): List of products being sold. Each product object contains:
    • productName (string, mandatory): Name of the product (max 36 characters).
    • unit (string, mandatory): Unit of measure (e.g., "buc", "kg", "L") (max 6 characters).
    • quantity (string, mandatory): Quantity sold. Numeric format with comma decimal separator (e.g., "1,000", "2,5"). Max 9 characters including comma.
    • pricePerUnit (string, mandatory): Price per unit. Numeric format with comma decimal separator (e.g., "10,50", "100,00"). Max 11 characters including comma.
    • vatId (string, mandatory): VAT identifier for this product (e.g., "A", "B", "I"). Must correspond to a pre-set VAT rate.
    • discountSign (string, optional): Sign for discount/surcharge on this item.
      • "0": Discount (value will be subtracted).
      • "1": Surcharge (value will be added).
    • discountType (string, optional, required if discountSign is present): Type of discount/surcharge.
      • "0": Amount-based.
      • "1": Percentage-based.
    • discountValue (string, optional, required if discountSign is present): Value of the discount/surcharge. Numeric format with comma decimal separator (e.g., "1,00" for amount, "10,00" for 10%). Max 9 characters including comma.
    • specialTaxName (string, optional): Name of any special tax applicable to this item (e.g., "Acciza") (max 16 characters).
    • specialTaxValue (string, optional, required if specialTaxName is present): Value of the special tax. Numeric format with comma decimal separator (e.g., "0,50"). Max 9 characters including comma.
  • payments (array of objects, mandatory): List of payment methods used. Each payment object contains:
    • paymentMethod (string, mandatory): Type of payment. Possible values:
      • "Cash"
      • "Card"
      • "Voucher"
      • "Credit" (On Account/Invoice)
      • "ModernPayment" (e.g., mobile payments, specific apps)
      • "Other"
      • "MealVoucher" (Bonuri de masa)
      • "GiftVoucher" (Tichete cadou)
      • "HolidayVoucher" (Tichete de vacanta)
      • "CulturalVoucher" (Tichete culturale)
      • "SocialVoucher" (Tichete sociale)
      • "NurseryVoucher" (Tichete de cresa)
    • amount (string, mandatory): Amount paid with this method. Numeric format with comma decimal separator (e.g., "125,50"). Max 11 characters including comma. The sum of amounts from all payment methods must equal the total sale amount.
  • cifNumber (string, optional): Customer's fiscal identification number (CIF/CUI) if applicable (max 16 characters).
  • adjustments (object, optional): Overall adjustments to the total sale amount. The keys are the VAT IDs ("A", "B", etc.) for which the adjustment applies. Each value is an object:
    • adjustmentSign (string, mandatory):
      • "-": Discount (value will be subtracted from the total for that VAT rate).
      • "+": Surcharge (value will be added to the total for that VAT rate).
    • adjustmentMode (integer, mandatory):
      • 0: Amount-based adjustment.
      • 1: Percentage-based adjustment (calculated on the total for that VAT rate).
    • adjustmentValue (string, mandatory): Value of the adjustment. Numeric format with comma decimal separator. Max 9 characters including comma.
  • freeTexts (array of objects, optional): List of free text lines to be printed on the receipt. Each object contains:
    • text (string, mandatory): The text to print (max 36 characters per line).
    • position (string, mandatory): See "Common Data Types". "head" or "foot".
{
"CashOperation": {
"type": "Add",
"amount": "500,00",
"description": "Depunere numerar sertar"
}
}

Command: CashOperation

Description: Registers a cash deposit (IN) or withdrawal (OUT) operation.

Parameters:

  • type (string, mandatory): Type of cash operation.
    • "Initial": Initial cash deposit at the beginning of the day/shift.
    • "Add": Cash deposit (IN).
    • "Withdraw": Cash withdrawal (OUT).
  • amount (string, mandatory): The amount of cash. Numeric format with comma decimal separator (e.g., "100,00"). Max 11 characters including comma.
  • description (string, mandatory): A brief description or reason for the operation (max 36 characters).
{
"NonFiscalReceipt": {
"textLines": [
{"text":"** TEST PRINT **","printAttributes":"CENTER,BOLD"},
{"text":"This is a non-fiscal receipt."},
{"text":"Line 3 with normal text."}
],
"hasHeader": true,
"hasFooter": false
}
}

Command: NonFiscalReceipt

Description: Prints a non-fiscal receipt, which does not affect fiscal totals. Useful for various informational printouts.

Parameters:

  • textLines (array of objects, mandatory): Lines of text to be printed. Each object has:
    • text (string, mandatory): The text content for the line (max 48 characters, but printer-dependent).
    • printAttributes (string, optional): See "Common Data Types". Comma-separated attributes like "CENTER", "BOLD".
  • hasHeader (boolean, optional, default: false): If true, prints the standard fiscal header (company info, etc.) before the custom text.
  • hasFooter (boolean, optional, default: false): If true, prints the standard fiscal footer (fiscal series, etc.) after the custom text.
