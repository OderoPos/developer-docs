Integrating with Remote Pay has two stages: a one-time device onboarding, and then sending commands. Both stages are the same for x30 and p2se devices.

1. Device onboarding (one-time, per device)

  1. Get credentials from us. For each device, Token provides you a set of sign-in credentials, issued per device.

  2. Sign in on the device. Open the Remote Pay app on the device (x30 or p2se) and log in with the credentials we provided for that device.

  3. The device configures itself. After sign-in, the device configures itself in the Token environment automatically. No further setup is required on your side.

Once onboarding is complete, the device is online and ready to receive your commands.

2. Sending commands

Before you can send commands, two things are arranged with us:

  • We assign you authentication credentials. Remote Pay is protected by OAuth 2.0. We issue you a dedicated client and the scopes you need. See Authentication.

  • You provide your callback URL(s). Command results are delivered to your callback URL: one default URL, or a different URL per device. This is the same for x30 and p2se. See Callbacks.

Then request your Postman collection from us. It is pre-filled with the endpoints and your environment so you can start immediately.

The flow of a single command:

  1. Your backend obtains an access token and calls the Remote Pay API for a specific device.
  2. Token securely delivers the command to that device.
  3. The device runs it: a card payment (x30 and p2se) or a fiscal command (x30).
  4. The result is delivered to your callback URL, the same for x30 and p2se.

The API call returns immediately to acknowledge that the command was accepted and delivered. The transaction result arrives later on your callback URL.
