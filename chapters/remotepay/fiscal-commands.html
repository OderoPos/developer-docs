Fiscal commands are an x30 capability. They run on the device fiscal core and produce a fiscal result, for example a sale receipt, a daily report, or device status.

The command bodies are identical to OPIT REST fiscal commands; Remote Pay relays them to the device. Use the same payloads documented under OPIT REST, Fiscal commands.

Make an authenticated POST request, where the last path segment is the fiscal command name:

POST https://{{baseUrl}}/x30tr/{deviceId}/{fiscalCommand}

Supported commands include saleRequest, getStatus, getDailyTotals, xReport, zReport, periodicReportByDate, periodicReportByZ, getCompanyData, setCompanyData, getVATRates, setVATRates, setDateTime, nonFiscalReceipt, cashOperation, getSoftwareVersion, and the journal and report commands. An unsupported command name returns 404.

The API responds immediately with a request id. The fiscal core result is delivered later to your callback URL, correlated by that request id. The full request and response are in your Postman collection.

Numeric formats (comma decimal separator) and field names follow the Romanian fiscal specification, the same as OPIT REST. For a card sale, the payment amount must match the receipt total exactly.
