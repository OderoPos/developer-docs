Remote Pay is asynchronous: the API call only confirms that a command was accepted and delivered to the device. The actual transaction result, the bank outcome of a payment or the fiscal core response of a fiscal command, is delivered later as an HTTP POST to a callback URL that you provide to us.

Callbacks apply to card payments on both x30 and p2se, and to fiscal results on x30. The delivery model is the same in all cases.

Registering your callback URL

  • You provide your callback URL(s) to Token during onboarding.
  • You can set one default URL for all of your devices, or a different URL per device. Per-device URLs are useful when each store or location receives results on its own endpoint.
  • The URL must be a publicly reachable HTTPS endpoint that accepts a POST with a JSON body.

What you receive

The callback body is the result object for the command. Use the id fields to correlate it with the request you made:

  • Payment results (x30 and p2se) carry payment_id and your order_id. See Payment command.
  • Fiscal results (x30) carry requestId. See Fiscal commands.

The full callback payloads are in your Postman collection.

Handling callbacks

  • Respond quickly with a 2xx status to acknowledge receipt.
  • Treat delivery as at-least-once: deduplicate on payment_id or requestId in case a callback is delivered more than once.
  • Correlate every callback with the id from the synchronous API response. Do not assume results arrive in the same order you sent the commands.
