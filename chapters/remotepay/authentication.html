Every Remote Pay API call must be authenticated. Remote Pay uses the OAuth 2.0 client credentials flow: your backend exchanges a client id and secret for a short-lived access token, then sends that token on every request.

We provide you, per integrator:

  • A dedicated client id and client secret.
  • The token URL.
  • The scope(s) your client may use, with separate credentials for the dev and prod environments.
  • A pre-configured Postman collection wired to these values, including the token request.

Request an access token from the token URL with grant type client_credentials and your scope, then send the returned token in the Authorization header on every call. The exact token request is in your Postman collection.

Notes:

  • Tokens are short-lived (about 1 hour). Cache and refresh before expiry, and request a new one on a 401.
  • Each method is restricted to a scope. A fiscal-scope token cannot call payment endpoints, and the reverse. Use the credentials that match the operation.
  • Keep the client secret on your server only. Never put it in a browser, a mobile app, or a shared Postman collection. Store it as a Postman secret variable.
