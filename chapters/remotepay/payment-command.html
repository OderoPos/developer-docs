The payment command triggers a card payment on the device. It is available on both x30 and p2se, and works identically on both: same request, same callback result.

Make an authenticated POST request to the pay endpoint for the target device. Only the device-family segment of the path differs:

  • x30: POST https://{{baseUrl}}/x30tr/{deviceId}/pay
  • p2se: POST https://{{baseUrl}}/p2se/{deviceId}/pay

Key fields:

  • deviceId (path): the target device id. x30 ids start with AV, p2se ids start with TJ.
  • order_id: your own reference for the sale, echoed back on the callback so you can correlate it.
  • amount: the amount to charge.
  • currency_code: the numeric ISO 4217 currency code, for example 946 for RON.

The API responds immediately with a payment id to confirm the command was accepted and delivered. The bank result is delivered later to your callback URL, for both x30 and p2se. See Callbacks.

The full request and response are in your Postman collection.
