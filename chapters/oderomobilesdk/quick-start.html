This guide takes you end to end through a single Odero payment on each supported platform — iOS, Android, and React Native. Every integration follows the same four steps: (1) your backend creates a payment session and returns its sessionId, (2) your app constructs the SDK, (3) your app calls showPaymentSheet with that sessionId, and (4) your app receives the result asynchronously.

The payment result is delivered through a notification (iOS), a callback (Android), or an event listener (React Native) — never as a return value or a completion closure of showPaymentSheet. Subscribe before you present the sheet.

Step 1 — Create the session on your backend (all platforms)

The payment session is always created server-side, on your own merchant backend — never inside the mobile app. The session holds the amount, currency, products, customer/billing visibility, and Apple Pay / Google Pay availability. Creating it on the server keeps your merchantApiToken off the device.

Your backend performs two calls against the Odero API (DEV base https://api-dev.pay.odero.ro/, PROD base https://pay.odero.ro/):

  1. POST util/merchant-signature-header-json — header X-Requested-With: XMLHttpRequest, body {merchantId, merchantApiToken, payload} → returns {merchant_signature}.
  2. POST api/payments/session — header X-MERCHANT-SIGNATURE: <merchant_signature>, body = the session payload → returns {sessionId}.

Your backend returns only the sessionId string to the app.

Session payload shape (sent by your backend to Odero). amount and product prices are in minor units (cents) — the sheet displays amount / 100. URL fields (extOrderUrl, returnUrl, imageUrl, …) are base64-encoded.

iOS

Requirements: iOS 15.0+, use_frameworks!, the two CocoaPods (OderoPaySDKIOS + its shared dependency), and an sdk.plist with ENV = DEV for testing. See the Installation chapter for pod setup.

Construct the SDK and observe results

Create one OderoPaySDK() instance and keep it for the lifetime of the screen. Results arrive on NotificationCenter under NSNotification.Name.paymentStatusUpdated, observed on the main queue.

Full minimal example

import UIKit
import OderoPaySDKIOS

final class CheckoutViewController: UIViewController {

    // Reuse a single instance.
    private let sdk = OderoPaySDK()

    override func viewDidLoad() {
        super.viewDidLoad()

        // 4) Subscribe BEFORE presenting the sheet.
        NotificationCenter.default.addObserver(
            self,
            selector: #selector(handlePaymentStatus(_:)),
            name: .paymentStatusUpdated,
            object: nil
        )
    }

    deinit { NotificationCenter.default.removeObserver(self) }

    // Tapped by your "Pay" button.
    @objc func pay() {
        // 1) Get the sessionId from YOUR backend (see Step 1).
        MyBackend.createSession { [weak self] sessionId in
            guard let self, let sessionId else { return }

            let savedCards: [SavedCardItem] = [
                SavedCardItem(
                    cardToken: "b9b94af4f9b2d221be6daaab8b68bc210289fcf1de8b195dda286bddc0cadf6c",
                    cvv: "123",
                    lastFourDigits: "8901",
                    expirationMonth: "06",
                    expirationYear: "2026"
                )
            ]

            // 3) Build the stock appearance — pass the seven default-constructed page
            // objects (iOS has no parameterless init / .default()).
            let appearance = OderoPaymentBottomSheetAppearance(
                oderoPaymentDetailsPageAppearance: OderoPaymentDetailsPageAppearance(),
                oderoPaymentProcessingPageAppearance: OderoPaymentProcessingPageAppearance(),
                oderoPaymentResultSuccessPageAppearance: OderoPaymentResultSuccessPageAppearance(),
                oderoPaymentResultFailurePageAppearance: OderoPaymentResultFailurePageAppearance(),
                oderoPaymentSummaryPageAppearance: OderoPaymentSummaryPageAppearance(),
                oderoPayWithAnotherCardPageAppearance: OderoPayWithAnotherCardPageAppearance(),
                oderoEnterCVCPageAppearance: OderoEnterCVCPageAppearance()
            )
            // Present the sheet on the main thread.
            DispatchQueue.main.async {
                self.sdk.showPaymentSheet(
                    sessionId: sessionId,
                    appearance: appearance,
                    applePayMerchantId: "merchant.token.applepay",
                    savedCards: savedCards
                )
            }
        }
    }

    @objc private func handlePaymentStatus(_ note: Notification) {
        let info = note.userInfo
        let status = info?["status"] as? String ?? ""

        switch status {
        case "payment_success":
            let ref = info?["referenceNumber"] as? String ?? ""
            print("Paid. Reference: \(ref)")
        case "payment_failure":
            let msg = info?["errorMessage"] as? String ?? "Unknown error"
            print("Failed: \(msg)")
        case "payment_processing", "payment_idle":
            break // transient states
        default:
            break
        }
    }
}

Notes

  • On iOS construct OderoPaymentBottomSheetAppearance with its seven default-constructed page objects (OderoPaymentDetailsPageAppearance() and six more) for the stock look — there is no parameterless initializer or .default() on iOS. Each page object is fully defaulted, so override one only to customize it.
  • applePayMerchantId must match the merchant id in your com.apple.developer.in-app-payments entitlement; the Apple Pay button only shows if the session enables it. Test on a real device.
  • Call sdk.closePaymentSheet() to dismiss programmatically. A user cancel simply dismisses the sheet (no dedicated event).

Android

Requirements: minSdk 23, compileSdk 35, Java 17 / Kotlin. The host must be a ComponentActivity (AppCompatActivity / FragmentActivity). Add com.google.android.gms:play-services-wallet for Google Pay. See the Installation chapter for the Gradle repo and dependency.

Construct the SDK and set the callback

Construct OderoPaySdk(activity, LOAD_PAYMENT_DATA_REQUEST_CODE, SCREEN_3DS_REQUEST_CODE) with request codes 1001 and 1002. Assign onPaymentStatusChanged to receive results.

Full minimal example

import android.os.Bundle
import androidx.appcompat.app.AppCompatActivity
import com.oderopaysdk.OderoPaySdk
import com.oderopaysdk.oderoui.models.SavedCardItem
import com.oderopaysdk.oderoui.paymentappearance.oderopaymentbottomsheetappearance.OderoPaymentBottomSheetAppearance
import kotlinx.coroutines.CoroutineScope
import kotlinx.coroutines.Dispatchers
import kotlinx.coroutines.launch

class CheckoutActivity : AppCompatActivity() {

    private lateinit var sdk: OderoPaySdk

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_checkout)

        // 2) Construct the SDK with the two request codes.
        sdk = OderoPaySdk(this, 1001, 1002)

        // 4) Set the result callback BEFORE presenting the sheet.
        sdk.onPaymentStatusChanged = { status, referenceNumber, errorMessage ->
            when (status) {
                "payment_success"  -> Log.d("Checkout", "Paid. Reference: $referenceNumber")
                "payment_failure"  -> Log.d("Checkout", "Failed: $errorMessage")
                else               -> Unit // payment_processing / payment_idle
            }
        }

        findViewById<View>(R.id.payButton).setOnClickListener { pay() }
    }

    private fun pay() = CoroutineScope(Dispatchers.Main).launch {
        // 1) Get the sessionId from YOUR backend (see Step 1).
        val sessionId: String = MyBackend.createSession()

        val savedCards = listOf(
            SavedCardItem(
                cardToken = "b9b94af4f9b2d221be6daaab8b68bc210289fcf1de8b195dda286bddc0cadf6c",
                cvv = "123",
                lastFourDigits = "8901",
                expirationMonth = "06",
                expirationYear = "26"
            )
        )

        // 3) Present the sheet. default() = stock look. Google Pay is automatic.
        sdk.showPaymentSheet(
            sessionId = sessionId,
            appearance = OderoPaymentBottomSheetAppearance.default(),
            savedCards = savedCards
        )
    }
}

Notes

  • OderoPaymentBottomSheetAppearance.default() gives the stock look; pass the seven page objects only to customize.
  • Google Pay is handled entirely by the SDK + session — there is no JS/Kotlin parameter. The button shows only when the session enables it and the device IsReadyToPay.
  • If your activity does its own 3DS result handling, forward it: in onActivityResult, when requestCode == 1002, call sdk.handle3DSScreenActivityResult(resultCode, data).
  • Call sdk.closePaymentSheet() to dismiss programmatically.

React Native

Requirements: a bare React Native app (not Expo Go); rebuild the app after installing react-native-odero-mobilesdk. iOS needs cd ios && pod install; Android needs the Odero Artifactory maven repo. See the Installation chapter for native setup.

Subscribe in useEffect, always .remove()

Import the named functions from react-native-odero-mobilesdk. Subscribe with addPaymentEventListener inside a useEffect and remove the subscription on cleanup. Results arrive only through this listener — showPaymentSheet returns nothing.

Full minimal example

import React, { useEffect } from 'react';
import { Button, View, Alert } from 'react-native';
import {
  showPaymentSheet,
  closePaymentSheet,
  addPaymentEventListener,
} from 'react-native-odero-mobilesdk';

export default function CheckoutScreen() {
  // 4) Subscribe once; remove on unmount.
  useEffect(() => {
    const subscription = addPaymentEventListener((event) => {
      if (event.status === 'success') {
        Alert.alert('Paid', 'Reference: ' + (event.referenceNumber ?? ''));
      } else if (event.status === 'failure') {
        Alert.alert('Failed', event.errorMessage ?? 'Unknown error');
      }
      // 'processing' | 'idle' are transient.
    });

    return () => subscription.remove(); // always clean up
  }, []);

  const pay = async () => {
    // 1) Get the sessionId from YOUR backend (see Step 1).
    const sessionId = await MyBackend.createSession();

    const savedCards = [
      {
        cardToken: 'b9b94af4f9b2d221be6daaab8b68bc210289fcf1de8b195dda286bddc0cadf6c',
        cvv: '123',
        lastFourDigits: '8901',
        expirationMonth: '06',
        expirationYear: '2026',
      },
    ];

    // 3) Present the sheet. appearance: {} = native defaults. No return value.
    showPaymentSheet(sessionId, {
      appearance: {},
      applePayMerchantId: 'merchant.token.applepay', // iOS only; ignored on Android
      savedCards,
    });
  };

  return (
    <View>
      <Button title="Pay" onPress={pay} />
      <Button title="Close" onPress={() => closePaymentSheet()} />
    </View>
  );
}

Notes

  • options is optional; pass appearance: {} for the native default look, or omit it entirely.
  • applePayMerchantId applies on iOS only (configure the entitlement + certificate in the iOS host, same as native iOS). Google Pay needs no JS parameter — it is handled by the native Android SDK and the session.
  • Always call subscription.remove() in the useEffect cleanup to avoid duplicate handlers across re-mounts.

Result reference

The result statuses are consistent across platforms; only the delivery mechanism differs.

PlatformDeliverySuccess carriesFailure carries
iOSNSNotification.Name.paymentStatusUpdated · userInfo["status"]userInfo["referenceNumber"]userInfo["errorMessage"]
AndroidonPaymentStatusChanged(status, referenceNumber, errorMessage)referenceNumbererrorMessage
React NativeaddPaymentEventListener(event => …)event.referenceNumberevent.errorMessage

Status values: iOS / Android use payment_idle, payment_processing, payment_success, payment_failure. React Native normalizes these to idle, processing, success, failure. Treat success and failure as terminal; the others are transient. A user-initiated cancel simply dismisses the sheet.

The DEV artifacts and sdk.plist ENV = DEV point at api-dev.pay.odero.ro and wallet TEST mode. Do not ship them to production.
