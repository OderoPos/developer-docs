This page lists the current published artifact versions of the Odero Mobile SDK for each platform, the OS/toolchain compatibility for those versions, and how the development and production artifacts differ.

Current artifact versions

The SDK is published as three independent platform artifacts. They are versioned separately — there is no single unified version number across platforms.

Platform Artifact Current version
iOS OderoPaySDKIOS (CocoaPods) 0.0.1
iOS (transitive) shared.xcframework (required dependency of OderoPaySDKIOS) 0.1.4
Android Odero.oderopaysdk:oderopaysdkandroid (Gradle/Maven) 1.0.0 (production) / 1.0.0-dev (development)
React Native react-native-odero-mobilesdk (npm) 0.1.1

The React Native package bridges the native iOS and Android SDKs. The native artifacts it pulls in (the iOS pod and the Android AAR) follow their own versions above.

Compatibility

The OS and toolchain requirements below apply to the current artifact versions.

Platform Minimum / required
iOS iOS 15.0+, Swift 5+, use_frameworks! (dynamic frameworks), CocoaPods
Android minSdk 23, compileSdk 35, Java 17 / Kotlin (AGP 8.x); host must be a ComponentActivity
React Native Bare React Native workflow (not Expo Go). iOS host: same as native iOS. Android host: compileSdk >= 37, minSdk 26, Kotlin Gradle Plugin pinned to 2.4.0

How versions map: development vs production

Each platform exposes a development variant that targets the Odero DEV environment (api-dev.pay.odero.ro) and wallet TEST mode, and a production variant that targets the live environment. The mechanism differs per platform.

  • iOS — the same pod version is used for both. The environment is selected at runtime by an sdk.plist in the app target: key ENV = DEV for development, or PROD (or omit the key) for production.
  • Android — the environment is baked into the artifact: use oderopaysdkandroid:1.0.0-dev for the development build (Google Pay TEST) or oderopaysdkandroid:1.0.0 for production. There is no runtime toggle.
  • React Native — selected at build time by the Oderomobilesdk_environment environment variable: DEV resolves the -dev Android artifact, anything else resolves the production artifact.

Development artifacts and the DEV sdk.plist point at api-dev.pay.odero.ro and use wallet TEST mode. Do not ship them to production.
