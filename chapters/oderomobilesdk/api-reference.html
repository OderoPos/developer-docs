The payment sheet's look and feel is fully data-driven. You describe the appearance with a tree of plain configuration objects and pass it to showPaymentSheet alongside the sessionId. There is no separate theming API to call and no global state to set — the appearance you pass to a given call styles that presentation of the sheet.

The object names are the same on iOS and Android. On React Native you provide the matching keys via the OderoAppearance object passed as options.appearance.

Everything is optional

Every field on every appearance object is optional and defaulted. To get the stock Odero theme, pass default-constructed page objects (iOS/Android) or an empty object / omit appearance entirely (React Native). Override only the handful of fields you care about and leave the rest untouched — the defaults fill in the rest.

Which sections appear in the sheet — billing fields, the Apple Pay / Google Pay button, the saved-cards list — is controlled by the payment SESSION created on your backend, not by appearance. Appearance only styles whatever the session decides to show; it cannot turn a section on or off.

Value types

All appearance values are passed as strings (plus a few booleans), which keeps the contract identical across native and React Native:

What you are settingValue formatExample
ColorsHex color string"#5A31F4"
Sizes, corner radii, border widthsNumeric string"16"
IconsBase64-encoded PNG string"iVBORw0KGgo..."
FontsFont-name string"DMSans-Bold"
Text / titles / placeholdersPlain string"Pay now"
Enabled / secure togglesBooleantrue

The appearance tree

OderoPaymentBottomSheetAppearance is the root object. It is composed of seven per-page appearance objects, one for each screen the sheet can present:

Page objectScreen it styles
oderoPaymentDetailsPageAppearanceThe main entry screen (largest — see below)
oderoPaymentProcessingPageAppearanceThe loading screen shown while the payment is processed
oderoPaymentResultSuccessPageAppearanceThe success result screen
oderoPaymentResultFailurePageAppearanceThe failure result screen
oderoPaymentSummaryPageAppearanceThe order/summary screen
oderoPayWithAnotherCardPageAppearanceThe "pay with another card" screen shown when saved cards exist
oderoEnterCVCPageAppearanceThe CVC-entry screen for a saved card

Each page object is itself all-defaulted: OderoPaymentDetailsPageAppearance(), OderoPaymentProcessingPageAppearance(), and so on (or the matching key under OderoAppearance on React Native).

The Details page is the largest

oderoPaymentDetailsPageAppearance covers far more surface than the other pages because it is where the customer actually enters payment information. Its fields style:

  • The payment-method selector at the top of the sheet
  • The saved-cards list (when the session provides saved cards)
  • The card fields — number, expiry, CVC — including their placeholder text and field icons
  • The billing fields (when the session enables billing)
  • The save-card checkbox
  • The bottom action buttons

The remaining six pages are small by comparison — typically a title, body text, an icon, background and button styling.

Examples

Each example below constructs the root appearance with stock pages and overrides only a couple of fields (a brand color and a title). Build it once and pass it on every showPaymentSheet call.

iOS (Swift)

import OderoPaySDKIOS

let appearance = OderoPaymentBottomSheetAppearance(
    oderoPaymentDetailsPageAppearance: OderoPaymentDetailsPageAppearance(
        payButtonBackgroundColor: "#5A31F4",   // brand color
        title: "Checkout"
    ),
    oderoPaymentProcessingPageAppearance: OderoPaymentProcessingPageAppearance(),
    oderoPaymentResultSuccessPageAppearance: OderoPaymentResultSuccessPageAppearance(),
    oderoPaymentResultFailurePageAppearance: OderoPaymentResultFailurePageAppearance(),
    oderoPaymentSummaryPageAppearance: OderoPaymentSummaryPageAppearance(),
    oderoPayWithAnotherCardPageAppearance: OderoPayWithAnotherCardPageAppearance(),
    oderoEnterCVCPageAppearance: OderoEnterCVCPageAppearance()
)

let sdk = OderoPaySDK()
sdk.showPaymentSheet(
    sessionId: sessionId,
    appearance: appearance,
    applePayMerchantId: "merchant.token.applepay",
    savedCards: []
)

Android (Kotlin)

import com.oderopaysdk.OderoPaymentBottomSheetAppearance
import com.oderopaysdk.OderoPaymentDetailsPageAppearance

val appearance = OderoPaymentBottomSheetAppearance(
    oderoPaymentDetailsPageAppearance = OderoPaymentDetailsPageAppearance(
        payButtonBackgroundColor = "#5A31F4",   // brand color
        title = "Checkout"
    ),
    oderoPaymentProcessingPageAppearance = OderoPaymentProcessingPageAppearance(),
    oderoPaymentResultSuccessPageAppearance = OderoPaymentResultSuccessPageAppearance(),
    oderoPaymentResultFailurePageAppearance = OderoPaymentResultFailurePageAppearance(),
    oderoPaymentSummaryPageAppearance = OderoPaymentSummaryPageAppearance(),
    oderoPayWithAnotherCardPageAppearance = OderoPayWithAnotherCardPageAppearance(),
    oderoEnterCVCPageAppearance = OderoEnterCVCPageAppearance()
)

// Or start from the stock theme and tweak: OderoPaymentBottomSheetAppearance.default()

sdk.showPaymentSheet(
    sessionId = sessionId,
    appearance = appearance,
    savedCards = emptyList()
)

React Native (TypeScript)

import { showPaymentSheet, OderoAppearance } from 'react-native-odero-mobilesdk';

const appearance: OderoAppearance = {
  oderoPaymentDetailsPageAppearance: {
    payButtonBackgroundColor: '#5A31F4',   // brand color
    title: 'Checkout',
  },
  // omit the other six pages to keep their stock look
};

showPaymentSheet(sessionId, {
  appearance,
  applePayMerchantId: 'merchant.token.applepay',
});

// Pass {} or omit `appearance` entirely for the fully default theme.

Full theme examples to copy

The example apps shipped with each SDK include complete custom themes that override colors, fonts, icons and titles across all seven pages. They are the fastest way to see every available field in context — copy one and adapt it to your brand rather than building an appearance tree from scratch.
