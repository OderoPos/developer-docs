Comprehensive error handling strategies for the Odero Mobile SDK.

Error Types

Network Errors

try {
  const result = await OderoSDK.processPayment(request);
} catch (error) {
  if (error.code === 'NETWORK_ERROR') {
    // Handle network connectivity issues
    showRetryDialog();
  }
}

Validation Errors

if (error.code === 'INVALID_AMOUNT') {
  // Handle invalid payment amount
  showErrorMessage('Please enter a valid amount');
}

Common Error Codes

  • NETWORK_ERROR: Network connectivity issues
  • INVALID_AMOUNT: Invalid payment amount
  • INSUFFICIENT_FUNDS: Insufficient funds on card
  • CARD_DECLINED: Card was declined
  • EXPIRED_CARD: Card has expired
  • INVALID_CREDENTIALS: Invalid API credentials

Retry Logic

const retryPayment = async (request, maxRetries = 3) => {
  for (let attempt = 1; attempt <= maxRetries; attempt++) {
    try {
      return await OderoSDK.processPayment(request);
    } catch (error) {
      if (attempt === maxRetries) throw error;
      await new Promise(resolve => setTimeout(resolve, 1000 * attempt));
    }
  }
};
