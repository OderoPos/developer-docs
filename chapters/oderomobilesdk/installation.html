This chapter walks through installing the Odero Pay SDK on each supported platform. All three SDKs resolve their binaries from Odero's Artifactory repository.

Each platform section is self-contained: pick the one you are integrating, follow it end to end, then continue to the configuration and quick-start chapters.

Backend vs. app. The payment session is always created on your merchant backend (server-side) using your merchantId + merchantApiToken. The app only ever receives a sessionId (a String) and passes it to the payment sheet. Never ship merchantApiToken inside the mobile app. See the Configuration chapter for the session contract.

iOS installation (CocoaPods)

The iOS SDK ships as the OderoPaySDKIOS module and is delivered as an XCFramework via CocoaPods. It depends on a second, shared XCFramework that the podspec pulls in transitively.

Requirements

  • iOS 15.0+, Swift 5+, Xcode with CocoaPods installed.
  • use_frameworks! (the SDK is distributed as dynamic frameworks).
  • PassKit (for Apple Pay) is linked automatically by the SDK.
  • The SDK bundles DM Sans fonts; register them at launch (covered in Configuration).

Two pods from Artifactory

Installation pulls two artifacts:

  1. The main pod OderoPaySDKIOS v0.0.1 — XCFramework zip:
    https://ro-artifactory.devtokeninc.com/artifactory/PublicLibraries/Odero/oderopaysdk/oderopaysdkios/0.0.1/oderopaysdkios-0.0.1.xcframework.zip
  2. The transitive shared core shared.xcframework v0.1.4 — XCFramework zip:
    https://ro-artifactory.devtokeninc.com/artifactory/PublicLibraries/Odero/oderopaysdk/oderopaysdkshared/shared.xcframework/0.1.4/shared.xcframework-0.1.4.zip

You declare a single pod; the podspec wires up the dependency on shared.xcframework for you. Downloads from Artifactory require the Accept: application/octet-stream header (CocoaPods sends this automatically when fetching the binary).

Podfile

Podfile — declare the platform, enable dynamic frameworks, add the pod

platform :ios, '15.0'
use_frameworks!

target 'YourApp' do
  # Pull OderoPaySDKIOS from the Odero-provided podspec.
  # The podspec resolves the OderoPaySDKIOS XCFramework (0.0.1) and its
  # transitive shared.xcframework (0.1.4) from Odero Artifactory.
  pod 'OderoPaySDKIOS', :podspec => '<odero-provided-podspec-url>'
end

Note. Replace <odero-provided-podspec-url> with the podspec URL Odero provides for your account. Do not point your Podfile directly at the raw zip URLs above — those are the artifacts the podspec downloads for you.

Install

pod install
# open the generated .xcworkspace (not the .xcodeproj) afterwards

Select the environment (sdk.plist)

The SDK reads its target environment from a property list named sdk.plist added to your app target. Set the ENV key to DEV for the development gateway, or use PROD (or omit the key entirely) for production.

sdk.plist — development

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>ENV</key>
    <string>DEV</string>
</dict>
</plist>

sdk.plist — production (or simply omit the ENV key)

<key>ENV</key>
<string>PROD</string>

Do not ship DEV to production. ENV = DEV points the SDK at the development gateway and enables Apple Pay test behavior. Ship PROD (or no ENV key) in your App Store builds.

Import

import OderoPaySDKIOS

Android installation (Gradle / Maven)

The Android SDK ships as an AAR in the com.oderopaysdk package, resolved from Odero's Artifactory Maven repository. Google Pay support requires the Google Play services Wallet dependency.

Requirements

  • minSdk 23, compileSdk 35.
  • Java 17 / Kotlin, AGP 8.x.
  • The host Activity that launches the sheet must be a ComponentActivity (e.g. AppCompatActivity or FragmentActivity).
  • Google Play services on the device for Google Pay.
  • The INTERNET permission is contributed by the SDK manifest — you do not need to add it yourself.

Add the Artifactory repository

Add the Odero Artifactory Maven URL to dependencyResolutionManagement in settings.gradle. The transitive shared core (Odero.oderopaysdk.oderopaysdkshared:shared-android) resolves from the same repository.

settings.gradle

dependencyResolutionManagement {
    repositoriesMode.set(RepositoriesMode.FAIL_ON_PROJECT_REPOS)
    repositories {
        google()
        mavenCentral()
        maven {
            url 'https://ro-artifactory.devtokeninc.com/artifactory/PublicLibraries/'
            // If your Artifactory instance is private, supply credentials:
            // credentials {
            //     username = providers.gradleProperty('oderoArtifactoryUser').get()
            //     password = providers.gradleProperty('oderoArtifactoryPassword').get()
            // }
        }
    }
}

Private repo? If access to PublicLibraries is restricted for your account, store the username/token in ~/.gradle/gradle.properties (never in version control) and wire them through the commented credentials block above.

Add the dependencies

Add the SDK and the Wallet dependency to your app-level build.gradle. Use the :1.0.0 coordinate for production; the :1.0.0-dev coordinate builds against the DEV gateway and Google Pay TEST environment. The environment is fixed by the artifact you choose — there is no runtime toggle.

app/build.gradle — production

dependencies {
    implementation 'Odero.oderopaysdk:oderopaysdkandroid:1.0.0'
    implementation 'com.google.android.gms:play-services-wallet:20.0.0'
}

app/build.gradle — development (DEV gateway + Google Pay TEST)

dependencies {
    implementation 'Odero.oderopaysdk:oderopaysdkandroid:1.0.0-dev'
    implementation 'com.google.android.gms:play-services-wallet:20.0.0'
}

app/build.gradle — compile/min SDK and Java 17

android {
    compileSdk 35

    defaultConfig {
        minSdk 23
    }

    compileOptions {
        sourceCompatibility JavaVersion.VERSION_17
        targetCompatibility JavaVersion.VERSION_17
    }

    kotlinOptions {
        jvmTarget = '17'
    }
}

Sync

./gradlew :app:dependencies --refresh-dependencies

ProGuard / R8. The AAR ships its own consumer rules; no extra keep rules are normally required.

React Native installation

The React Native package is react-native-odero-mobilesdk (v0.1.1). It bridges the native iOS and Android SDKs, so the native setup steps above still apply to each platform underneath.

Package name correction. Older docs referenced @odero/react-native-sdk. That package does not exist — the real, published package is react-native-odero-mobilesdk.

Requirements

  • Bare React Native — Expo Go is not supported (the SDK contains native code). A prebuilt/bare workflow is required, and you must rebuild the app after installing.
  • iOS: 15.0+, CocoaPods.
  • Android: compileSdk >= 37, minSdk 26, Kotlin Gradle Plugin pinned to 2.4.0.

Add the package

# npm
npm install react-native-odero-mobilesdk

# or yarn
yarn add react-native-odero-mobilesdk

package.json

{
  "dependencies": {
    "react-native-odero-mobilesdk": "0.1.1"
  }
}

Select the environment

The native build picks its gateway from the Oderomobilesdk_environment environment variable: set it to DEV to build against the DEV artifacts (development gateway + wallet TEST); any other value (or unset) builds for PROD.

# Development build (DEV gateway, wallet TEST)
export Oderomobilesdk_environment=DEV

# Production build — leave unset, or:
export Oderomobilesdk_environment=PROD

iOS native setup

From the project root, install pods. The podspec pulls OderoPaySDKIOS from the Odero podspec source (the same Artifactory-backed XCFrameworks described in the iOS section).

cd ios && pod install

Two iOS gotchas on recent Xcode toolchains:

  1. Nested framework re-sign. CocoaPods embeds OderoPaySDKIOS, which itself contains a nested shared.framework. Add a re-sign build phase that code-signs the nested framework after CocoaPods' "Embed Frameworks" phase, otherwise the app fails to launch/upload with a signing error.
  2. fmt / consteval patch. On recent Xcode, add an fmt consteval patch in your Podfile post_install hook to fix a compile error in the fmt transitive dependency.

Podfile post_install — pointer for the fmt patch (apply Odero's provided snippet)

post_install do |installer|
  react_native_post_install(installer)
  # Apply the Odero-provided fmt consteval patch here on recent Xcode,
  # and add a build phase that re-signs the nested shared.framework.
end

Android native setup

Add the Odero Artifactory Maven repository (so Odero.oderopaysdk:oderopaysdkandroid resolves), bump compileSdk to 37+, keep minSdk at 26, and pin the Kotlin Gradle Plugin to 2.4.0.

android/build.gradle — pin Kotlin and add the repository

buildscript {
    ext {
        compileSdkVersion = 37
        minSdkVersion = 26
        kotlinVersion = '2.4.0'
    }
}

allprojects {
    repositories {
        google()
        mavenCentral()
        maven {
            url 'https://ro-artifactory.devtokeninc.com/artifactory/PublicLibraries/'
        }
    }
}

Autolinking wires the native module in for you; no manual linking is required.

INTERNET permission

android/app/src/main/AndroidManifest.xml

<uses-permission android:name="android.permission.INTERNET" />

Import

import {
  showPaymentSheet,
  closePaymentSheet,
  addPaymentEventListener,
} from 'react-native-odero-mobilesdk';

Rebuild after installing. Because the package ships native code, run a fresh native build (npx react-native run-ios / run-android) after adding it — a Metro reload is not enough.
